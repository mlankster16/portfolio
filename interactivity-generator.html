Interactivity Generator

A two-part workflow tool built for Duke's LX Design team

This one is different. Instead of designing a course, I designed and built a tool that helps our team design courses.

I kept running into the same limit

Throughout our Coursera courses, we've always looked for new ways to help learners access and explore content. For a long time, we were limited by the tools available to us. Interactive formats we wanted to build had no clear path into the Coursera environment, so good ideas stayed ideas.

With what AI makes possible now, I decided to build our own.

I built it in two parts

First, a Coursera plugin. Then, a generator that our own LX design team could use on their own projects.

Part 1

The Coursera Plugin

Takes HTML and JSON and embeds it directly into the Coursera course environment.

Part 2

The Interactivity Generator

Turns a learning design challenge into a set of interactive format options, ready to edit and export.

A designer describes a learning design challenge from a project they're working on, and the tool proposes a set of interactive formats that could fit it. They pick one, edit the learner-facing text on a dedicated edit screen, and confirm it. The tool then renders the HTML and JSON, ready to paste straight into the plugin.

I'll show you how it works

The real tool calls an AI model live and works from whatever content a designer brings to it. What follows is a mockup built around one fixed example, so you can walk through the same four steps without needing an API key of your own.

Why this one matters to me

What I like about this project is that it proves two things at once: the learning design judgment to know what kind of interactive actually helps, and the technical skill to build a tool that generates it.

This was built for Coursera and for Duke's specific needs, but the underlying skill set travels. I'm looking for new places to put it to work.
