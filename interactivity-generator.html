Interactivity Generator

A two-part workflow tool built for Duke's LX Design team

This one is different. Instead of designing a course, I designed and built a tool that helps our team design courses.

I noticed a gap in what we could build quickly

Coursera is well suited to heavier interactive formats like Labs, Dialogue, and Role Play. What it doesn't have is a lightweight path for quick, embedded interactions, the kind that turn a single concept into a fast knowledge check or an exploratory visual. Without that path, those concepts default to static text, and learners lose the low-stakes moment to test their understanding right when a concept is introduced. I design interactives often, across a range of learning environments, some of that work lives under Sample Learning Assets, and I kept feeling that gap specifically inside Coursera.

This project set out to close it: give our learning designers a way to turn raw content into custom, embeddable interactions, faster and with a better pedagogical fit than we had before, with accessibility and UDL built in from the start rather than added on later.

I built it in two phases

First, a Coursera plugin that takes finished HTML and JavaScript and embeds it directly into the course environment. Then, a generator our own LX design team could use on their own projects.

Phase 1

The Coursera Plugin

Takes finished HTML and JavaScript and embeds it directly into the Coursera course environment.

Phase 2

The Interactivity Generator

Turns a learning design challenge into a set of interactive format options, ready to edit and export.

A designer describes their content and learning goal, or picks the interaction type themselves in "I know what I want" mode. The tool uses Claude to recommend the strongest-fit format from a curated set, tabbed explorer, click-to-reveal, inline quiz, sequencing, drag-to-classify, or annotated text, grounded in learning design decision rules. It also flags common missteps: a challenge that's out of scope, too content-heavy for one interaction, too vague to act on, or mismatched to the format chosen.

I'll show you how it works

From there, the designer gets a live, working preview and can explore alternatives, edit the visible text without touching code, and confirm. The tool outputs a ready-to-paste plugin configuration and an accessible text version. Our LX design team in CTL are the primary users. SMEs review the result while or after a designer generates a component, to confirm the content itself holds up.

The real tool calls Claude live and works from whatever content a designer brings to it. What follows is a mockup built around one fixed example, so you can walk through the same four steps without needing an API key of your own.

It also generates a static twin

Alongside the interactive component, the tool generates a plain static version of the same content. That does two jobs at once.

For learners, it's a second way in: a study reference, or the primary way to access the content if the interactive itself isn't accessible to them, by screen reader or otherwise.

For Coursera itself, it gives AI-enabled features like Dialogue something to work with. Those tools rely on course content to operate, and static text is what they can read when the interactive can't be parsed directly.
